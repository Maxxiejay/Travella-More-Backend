Authentication API Endpoints

The authentication API provides endpoints for user registration, login, profile management, and email verification.

Endpoint Method Description Access
/api/auth/signup POST Register a new user Public
/api/auth/signin POST Authenticate user & get token Public
/api/auth/profile GET Get user profile data Private
/api/auth/verify-email/:token GET Verify email with token Public
/api/auth/resend-verification POST Resend email verification Public
/api/auth/forgot-password POST Request password reset email Public
/api/auth/reset-password/:token GET Validate password reset token Public
/api/auth/reset-password/:token POST Reset password with token Public
/api/auth/test-email POST Send a test email to verify email configuration Public
Authentication

Private endpoints require a valid JWT token in the Authorization header:

Authorization: Bearer <token>

API Testing

You can test the API endpoints using the following tools:

cURL

Test API endpoints from the command line

curl -X POST http://localhost:8000/api/auth/signup \
-H "Content-Type: application/json" \
-d '{"username": "test", "email": "test@example.com", "password": "Test123"}'
Postman

Use Postman for a graphical interface to test API endpoints

Simply import the endpoint URLs and test with various HTTP methods
